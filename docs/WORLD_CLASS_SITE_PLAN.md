# World-Class Site Plan — ScanAdmin

## Ambition

Construire un site de niveau mondial pour ScanAdmin : crédible, rapide, clair, orienté conversion, SEO, confiance et future scalabilité SaaS.

ScanAdmin ne doit pas ressembler à une petite landing page bricolée. Le site doit donner l'impression d'un vrai produit sérieux, capable de devenir une référence internationale du scan administratif assisté par IA.

## Positionnement global

ScanAdmin est le scan IA administratif qui aide les PME à comprendre combien leurs tâches répétitives leur coûtent chaque mois, puis à identifier les agents IA supervisés les plus rentables à mettre en place.

## Promesse principale

Découvrez combien vos tâches administratives répétitives vous coûtent chaque mois, et quels agents IA peuvent vous faire gagner du temps et de l'argent.

## Standard attendu

Le site doit être :

- très clair en moins de 5 secondes ;
- parfaitement lisible sur mobile ;
- crédible pour un dirigeant de PME ;
- rassurant sur les limites de l'IA ;
- orienté capture de lead ;
- optimisé SEO ;
- rapide sur Vercel ;
- prêt pour l'internationalisation ;
- prêt pour une future monétisation.

## Architecture produit cible

### Pages principales

- `/` : accueil commerciale.
- `/scan` : formulaire de scan.
- `/result` : résultat du scan.
- `/analyse` : demande d'analyse personnalisée.
- `/merci` : confirmation de demande.
- `/tarifs` : offres commerciales.
- `/agents` : catalogue des agents IA supervisés.
- `/rapport-exemple` : exemple de rapport ScanAdmin.
- `/methodologie` : explication du modèle de calcul.
- `/confidentialite` : confidentialité et RGPD.
- `/mentions-legales` : mentions légales.

### Pages SEO prioritaires

- `/audit-ia-administratif-pme`
- `/calculateur-cout-administratif`
- `/agent-ia-devis`
- `/agent-ia-relance-client`
- `/agent-ia-documents`
- `/automatisation-administrative-pme`
- `/scan-ia-pme`
- `/ia-pour-administratif-pme`

## Accueil world-class

La page d'accueil doit contenir :

1. Hero section très claire.
2. CTA principal : lancer le scan gratuit.
3. CTA secondaire : voir un exemple de résultat.
4. Bloc problème : l'administratif coûte plus cher qu'on ne le croit.
5. Bloc calcul : temps perdu, coût mensuel, coût annuel.
6. Bloc agents IA supervisés : l'IA prépare, l'humain valide.
7. Bloc exemples de tâches : mails, devis, relances, factures, documents, dossiers clients.
8. Bloc résultat type.
9. Bloc offres : scan gratuit, analyse personnalisée, agent IA supervisé.
10. Bloc confiance : estimation indicative, pas de promesse de ROI garanti.
11. FAQ.
12. CTA final.

## Page résultat world-class

La page résultat doit expliquer :

- ce que signifie le temps administratif estimé ;
- comment le coût mensuel est calculé ;
- pourquoi le coût annuel est important ;
- ce que signifie le temps récupérable ;
- pourquoi les agents sont recommandés ;
- quelles sont les limites de l'estimation ;
- quelle est la prochaine étape.

Elle doit inclure un CTA fort :

> Recevoir mon analyse personnalisée

## Capture lead

Le bouton d'analyse ne doit pas rester un simple `mailto:`.

Objectif V2 : créer un vrai formulaire avec :

- prénom ;
- nom ;
- entreprise ;
- email ;
- téléphone optionnel ;
- secteur ;
- résultat du scan ;
- message optionnel ;
- consentement RGPD simple.

Le formulaire doit mener vers `/merci`.

## Confiance et conformité

Le site doit préciser :

- les résultats sont des estimations ;
- les gains réels dépendent des processus internes ;
- aucune automatisation n'est déployée sans validation humaine ;
- les agents IA sont supervisés ;
- les données ne doivent pas être revendues ;
- une politique de confidentialité est nécessaire avant acquisition réelle.

## SEO

Chaque page doit avoir :

- title unique ;
- meta description ;
- H1 clair ;
- structure H2/H3 propre ;
- contenu long utile ;
- liens internes vers `/scan` et `/analyse` ;
- FAQ ;
- mots-clés naturels.

Mots-clés prioritaires :

- scan IA administratif ;
- audit IA administratif PME ;
- calculateur coût administratif ;
- automatisation administrative PME ;
- agent IA devis ;
- agent IA relance client ;
- agent IA documents ;
- tâches administratives répétitives ;
- gain de temps administratif.

## UX mobile

Priorité absolue : mobile first.

Règles :

- titres lisibles sans être trop massifs ;
- boutons visibles ;
- cartes espacées ;
- résultats compréhensibles en scroll vertical ;
- formulaire rapide ;
- pas de champs inutiles ;
- CTA toujours évident.

## Design

Le design doit rester :

- sobre ;
- premium ;
- professionnel ;
- moderne ;
- lisible ;
- cohérent avec une cible PME.

Éviter :

- animations inutiles ;
- jargon startup ;
- promesses agressives ;
- couleurs trop flashy ;
- site trop vide.
