# ScanAdmin — plan SEO + AI-first

## Objectif

La stratégie ScanAdmin n’est pas de chercher activement des testeurs.

La stratégie est :

```txt
être trouvé par les dirigeants de PME
au moment où ils cherchent à comprendre
combien leur administratif leur coûte
et quoi automatiser avec l’IA en premier.
```

Le site doit être pensé pour :

- Google ;
- Bing ;
- ChatGPT Search ;
- moteurs IA ;
- utilisateurs qui posent des questions longues ;
- pages facilement citables par une IA.

## Positionnement SEO central

ScanAdmin ne doit pas se positionner comme un simple outil IA.

Positionnement :

```txt
Diagnostic du coût administratif répétitif des PME
+ priorisation des agents IA supervisés à tester.
```

Phrase de référence :

```txt
ScanAdmin aide les PME à estimer le coût mensuel de leurs tâches administratives répétitives et à identifier le premier agent IA supervisé à tester.
```

## Principe AI-first

Une page AI-first doit pouvoir être comprise, résumée et citée facilement.

Chaque page importante doit contenir :

- une réponse claire dès le début ;
- des définitions simples ;
- des exemples concrets ;
- des limites ;
- une FAQ ;
- des tableaux ;
- des phrases autonomes ;
- un lien vers le scan gratuit ;
- un lien vers l’analyse personnalisée.

À éviter :

- jargon IA inutile ;
- promesse de gain garanti ;
- contenu trop marketing ;
- pages trop courtes ;
- répétition artificielle de mots-clés.

## Architecture de contenu cible

Objectif à terme : 30 à 50 pages fortes, pas 300 pages faibles.

Familles :

```txt
1. guides généraux
2. pages tâches administratives
3. pages agents IA
4. pages secteurs
5. comparatifs
6. pages objections / confiance
7. FAQ longues
```

## Robots et crawlers IA

Objectif : ne pas bloquer les crawlers utiles à la découverte.

À vérifier dans `robots.txt` :

```txt
Googlebot autorisé
Bingbot autorisé
OAI-SearchBot autorisé si souhaité
ChatGPT-User autorisé si souhaité
sitemap déclaré correctement
```

Règle : ne pas ouvrir les pages tunnel au référencement.

Pages à garder hors index :

```txt
/result
/analyse
/merci
/paiement/succes
/paiement/retour
```

## Critères de réussite SEO

À 90 jours, résultat encourageant :

```txt
20+ pages indexées
impressions régulières
clics organiques chaque semaine
plusieurs scans issus du SEO
au moins un paiement ou une demande sérieuse
```

Résultat moyen :

```txt
pages indexées
impressions faibles
peu de clics
aucun paiement
```

Action : renforcer les titres, exemples, FAQ, pages tâches et preuve du livrable.

Résultat mauvais :

```txt
pages non indexées
aucune impression
aucun scan
```

Action : revoir sitemap, robots, qualité des pages, maillage interne et Search Console.


Chaque page tâche doit pointer vers :

```txt
agent correspondant

```

## Règle éditoriale absolue

Toujours répéter la nuance stratégique :

```txt
L’IA prépare, l’humain valide.
```

Ne jamais dire :

```txt
l’IA remplace votre administratif
l’IA garantit un gain
l’IA envoie vos devis automatiquement
l’IA décide à votre place
```

## Décision stratégique

ScanAdmin ne doit pas chercher la croissance par prospection au départ.

La croissance recherchée est :

```txt
SEO long terme
+ pages très ciblées
+ confiance
+ exemples concrets
+ conversion scan → analyse payante
```

Le signal de réussite devient :

```txt
Est-ce que des visiteurs organiques comprennent le problème,
lancent le scan,
puis commandent une analyse personnalisée ?
```
