# ScanAdmin — modèles de messages client

## Objectif

Ce document sert à répondre proprement aux clients après :

```txt
paiement de l’analyse personnalisée à 200 €
demande d’informations complémentaires
livraison du livrable
demande de mise en place d’un agent IA
relance qualité
```

Règle officielle :

```txt
TimeProofs exploite ScanAdmin.
ScanAdmin vend une analyse personnalisée à 200 €.
La mise en place d’un agent IA n’est pas incluse dans ces 200 €.
Toute création, configuration ou intégration IA fait l’objet d’un cadrage et d’un devis séparé.
```

---

## 1. Message automatique après paiement confirmé

Objet :

```txt
Votre analyse personnalisée ScanAdmin est bien confirmée
```

Message :

```txt
Bonjour [Prénom / Nom],

Votre paiement pour l’analyse personnalisée ScanAdmin a bien été confirmé.

Nous allons maintenant vérifier les informations transmises lors de votre scan afin de préparer votre analyse.

Votre analyse comprendra notamment :

- une lecture de vos tâches administratives répétitives ;
- une estimation des priorités ;
- l’identification du premier cas d’usage IA supervisé à tester ;
- les limites et points de vigilance ;
- un plan de test simple et prudent.

Important : l’analyse personnalisée ne comprend pas la création, l’installation ou le paramétrage complet d’un agent IA. Si vous souhaitez ensuite aller plus loin, nous pourrons cadrer ce besoin séparément.

Délai indicatif de livraison : 48 à 72 h ouvrées après réception des informations nécessaires.

Cordialement,
Jeason Bacoul
ScanAdmin — service exploité par TimeProofs
```

Objet :

```txt

```

```txt
Bonjour [Prénom / Nom],

Merci pour votre demande d’analyse personnalisée ScanAdmin.

Avant de finaliser votre analyse, j’ai besoin de quelques précisions afin d’éviter une recommandation trop vague ou mal adaptée à votre situation.

Pouvez-vous me répondre simplement à ces questions ?

1. Quelle est votre activité principale ?
2. Quelle tâche administrative vous prend le plus de temps aujourd’hui ?
3. Combien de fois par semaine cette tâche revient-elle environ ?
4. Qui valide aujourd’hui cette tâche dans votre entreprise ?
5. Utilisez-vous déjà un outil pour les emails, devis, documents, factures ou planning ?
6. Y a-t-il une tâche que vous ne voulez surtout pas confier à l’IA ?

Dès réception de ces éléments, je finalise l’analyse.

Cordialement,
Jeason Bacoul
```

```txt

- les données nécessaires ;

- les limites de ce que l’agent ne doit pas faire ;

- le format attendu du résultat.

Jeason Bacoul
```

## 5. Questions de cadrage si le client veut une IA

Ne pas proposer un prix immédiatement.

Commencer par ces questions :

```txt
1. Quelle tâche voulez-vous traiter en premier ?
2. Quel résultat attendez-vous exactement de l’agent IA ?
3. Où arrivent les informations aujourd’hui ? Email, formulaire, Drive, CRM, logiciel métier ?
4. Qui valide actuellement cette tâche ?
5. Qu’est-ce que l’IA ne doit jamais faire seule ?
6. Combien de fois par semaine cette tâche revient-elle ?
7. Quels outils doivent rester dans le processus ?
8. Faut-il connecter l’agent à un outil existant ?
9. Avez-vous des exemples réels anonymisables ?
10. Quel serait un test réussi après 14 jours ?
```

```txt
Si vous le souhaitez, je peux vous proposer un cadrage séparé pour cette mise en place.
```

## 11. Règle commerciale interne

Ne jamais dire :

```txt
on va tout automatiser
l’agent IA va remplacer votre administratif
l’IA va gérer vos clients seule
vous allez forcément gagner X heures
le gain est garanti
c’est inclus dans les 200 €
```

Dire plutôt :

```txt
on identifie le premier cas utile
on commence par un test supervisé
l’IA prépare, l’humain valide
le gain doit être mesuré
la mise en place IA fait l’objet d’un devis séparé
```
