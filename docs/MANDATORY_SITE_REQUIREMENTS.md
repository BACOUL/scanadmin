# Exigences obligatoires et indispensables — ScanAdmin

Ce document liste ce qu'un site sérieux doit avoir avant d'être considéré comme crédible, exploitable commercialement et prêt à recevoir du trafic.

## 1. Pages légales obligatoires

### Mentions légales

Page à créer :

```txt
/mentions-legales
```

Contenu minimal à prévoir :

- identité de l'éditeur ;
- statut juridique ;
- adresse ou siège ;
- email de contact ;
- téléphone si applicable ;
- directeur de publication ;
- hébergeur du site ;
- informations d'immatriculation si société ou entrepreneur individuel ;
- TVA si applicable.

### Politique de confidentialité

Page à créer :

```txt
/confidentialite
```

Contenu minimal à prévoir :

- données collectées ;
- finalité de la collecte ;
- base légale ;
- durée de conservation ;
- destinataires des données ;
- droits des utilisateurs ;
- contact pour exercer les droits ;
- mention CNIL ;
- transfert hors UE si applicable.

### Cookies

À prévoir seulement si des cookies ou traceurs non strictement nécessaires sont utilisés.

Page ou section à prévoir :

```txt
/cookies
```

Règle : ne pas installer de traceur marketing sans consentement explicite.

## 2. Pages de confiance indispensables

### Méthodologie

Page à créer :

```txt
/methodologie
```

Objectif : expliquer comment les calculs sont faits.

Contenu :

- formule temps perdu ;
- formule coût mensuel ;
- formule coût annuel ;
- taux d'automatisation par tâche ;
- limites de l'estimation ;
- aucune garantie de ROI sans audit.

### Exemple de rapport

Page à créer :

```txt
/rapport-exemple
```

Objectif : montrer ce que l'utilisateur peut recevoir après une analyse personnalisée.

### Catalogue agents IA

Page à créer :

```txt
/agents
```

Objectif : montrer les agents IA supervisés proposés.

Agents prioritaires :

- agent mails ;
- agent devis ;
- agent relance ;
- agent documents ;
- agent factures ;
- agent dossiers clients ;
- agent reporting.

### Tarifs

Page à créer :

```txt
/tarifs
```

Objectif : rendre l'offre compréhensible.

Offres cibles :

- scan gratuit ;
- analyse personnalisée ;
- audit administratif IA ;
- installation agent IA supervisé ;
- maintenance.

## 3. Pages de conversion indispensables

### Analyse personnalisée

Page existante ou à améliorer :

```txt
/analyse
```

Elle doit récupérer le résultat du scan et envoyer une demande exploitable.

### Merci

Page existante ou à améliorer :

```txt
/merci
```

Elle doit confirmer la demande et indiquer la suite.

### Contact

Page à créer :

```txt
/contact
```

Objectif : permettre à quelqu'un de contacter ScanAdmin sans passer par le scan.

## 4. Éléments UX obligatoires

- header clair ;
- footer complet ;
- navigation visible ;
- CTA principal cohérent ;
- CTA secondaire ;
- site mobile first ;
- page résultat compréhensible ;
- avertissement visible sur les estimations ;
- accès facile aux pages légales.

## 5. SEO obligatoire

Chaque page doit avoir :

- titre SEO unique ;
- meta description ;
- H1 unique ;
- H2 structurés ;
- liens internes ;
- contenu utile ;
- CTA vers le scan ;
- URL propre.

À prévoir :

- sitemap ;
- robots.txt ;
- Open Graph ;
- favicon ;
- canonical ;
- indexation propre.

## 6. Performance et qualité

Objectif : site rapide et fiable.

À vérifier :

- build Next.js sans erreur ;
- responsive mobile ;
- accessibilité des formulaires ;
- champs obligatoires clairs ;
- messages d'erreur lisibles ;
- pas de page vide ;
- pas de lien mort ;
- pas de CTA non fonctionnel.

## 7. Analytics et suivi

À prévoir avant acquisition réelle :

- suivi des visites ;
- clics sur CTA ;
- scans commencés ;
- scans terminés ;
- demandes d'analyse ;
- secteurs des prospects ;
- agents recommandés les plus fréquents.

## 8. Données et sécurité

À prévoir :

- ne pas exposer les données personnelles ;
- ne pas stocker inutilement ;
- limiter les accès ;
- clarifier la durée de conservation ;
- prévoir suppression sur demande ;
- sécuriser l'envoi des leads ;
- ne pas envoyer de données sensibles dans les logs.

## 9. Crédibilité commerciale

Un site top mondial doit montrer :

- promesse claire ;
- problème concret ;
- résultat exemple ;
- méthodologie ;
- agents proposés ;
- limites assumées ;
- tarifs ou offres ;
- FAQ ;
- preuve ou cas d'usage ;
- contact réel.

## 10. Priorité d'exécution

Ordre recommandé :

1. Footer + navigation.
2. Mentions légales.
3. Confidentialité.
4. Méthodologie.
5. Page agents.
6. Page tarifs.
7. Exemple de rapport.
8. Contact.
9. Sitemap + robots.
10. SEO par page.
11. Analytics.
12. Connexion réelle des leads.

## Règle finale

ScanAdmin ne doit pas seulement être beau. Il doit être :

- légalement propre ;
- commercialement clair ;
- techniquement fiable ;
- crédible pour une PME ;
- prêt à recevoir du trafic ;
- prêt à convertir un visiteur en prospect.
