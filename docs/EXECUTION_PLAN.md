# Plan d'exécution complet — ScanAdmin

Ce document sert à reprendre le projet de A à Z sans perdre le contexte.

## Phase 0 — Base projet

Objectif : avoir un repo clair et déployable.

À faire :

- repo GitHub ;
- README complet ;
- documentation projet ;
- structure Next.js ;
- Vercel ;
- domaine `scanadmin.fr` si disponible ;
- première landing page.

Critère de réussite : le site affiche une page d'accueil claire.

## Phase 1 — MVP scan gratuit

Objectif : permettre à un visiteur de calculer ses pertes administratives.

À faire :

- formulaire simple ;
- calcul du temps administratif mensuel ;
- calcul du coût mensuel ;
- calcul du coût annuel ;
- estimation du temps récupérable ;
- recommandation des 3 agents IA prioritaires ;
- page résultat lisible mobile.

Critère de réussite : un utilisateur peut faire le scan sans compte et obtenir un résultat.

## Phase 2 — Capture commerciale

Objectif : transformer le scan en prospect.

À faire :

- formulaire de demande d'analyse ;
- email de réception ;
- sauvegarde des résultats ;
- page merci ;
- notification à Jeason ;
- éventuellement export CSV ou Google Sheet.

Critère de réussite : chaque scan intéressé crée un lead exploitable.

## Phase 3 — Rapport personnalisé

Objectif : vendre une analyse express.

À faire :

- modèle de rapport PDF ;
- score administratif ;
- tâches les plus coûteuses ;
- agents recommandés ;
- estimation basse et haute ;
- prix de l'analyse personnalisée ;
- prise de rendez-vous.

Critère de réussite : pouvoir vendre une analyse entre 290 et 490 euros.

## Phase 4 — Offres payantes

Objectif : monétiser le projet.

Offres recommandées :

| Offre | Prix cible | But |
|---|---:|---|
| Scan gratuit | 0 euro | acquisition |
| Analyse personnalisée | 290 à 490 euros | qualification et cash |
| Audit administratif IA | 990 à 1 990 euros | mission conseil |
| Agent IA supervisé | 1 500 à 4 000 euros | prestation principale |
| Maintenance | 250 à 800 euros par mois | revenu récurrent |

## Phase 5 — Agents IA supervisés

Objectif : créer des modèles d'agents réutilisables.

Agents prioritaires :

1. Agent mails.
2. Agent devis.
3. Agent relance.
4. Agent documents.
5. Agent dossiers clients.
6. Agent reporting.

Chaque agent doit avoir :

- objectif ;
- données nécessaires ;
- limites ;
- validation humaine ;
- exemple d'usage ;
- estimation de gain ;
- procédure d'installation.

## Phase 6 — Produit SaaS

Objectif : transformer l'offre en produit scalable.

À faire plus tard :

- comptes utilisateurs ;
- historique des scans ;
- tableau de bord ;
- rapport PDF automatique ;
- catalogue d'agents activables ;
- paiement Stripe ;
- emails transactionnels ;
- espace admin ;
- analytics.

## Priorité actuelle

Ne pas passer à la Phase 6 trop tôt.

La priorité est :

1. MVP fonctionnel.
2. Déploiement Vercel.
3. Test auprès de PME.
4. Récupération de leads.
5. Vente des premiers audits.
