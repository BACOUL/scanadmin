# ScanAdmin — process demande de mise en place d’un agent IA

## Objectif

Définir quoi faire si un client, après son analyse ScanAdmin, souhaite réellement mettre en place un agent IA.

Règle centrale :

```txt
L’analyse à 200 € recommande un agent IA supervisé.
Elle ne comprend pas la création ou l’installation complète de cet agent.
```

Toute mise en place d’agent IA est une prestation séparée, cadrée et facturée à part par TimeProofs / ScanAdmin.

---

## Positionnement

ScanAdmin ne vend pas une automatisation magique.

ScanAdmin vend une démarche prudente :

```txt
1. comprendre la charge administrative ;
2. choisir une tâche prioritaire ;
3. vérifier que l’IA est pertinente ;
4. cadrer un test supervisé ;
5. déployer seulement si le risque est maîtrisé.
```

---

## Ce qui déclenche ce process

Le process démarre si le client dit par exemple :

```txt
Je veux mettre en place l’agent IA recommandé.
Vous pouvez nous le créer ?
Vous pouvez connecter ça à nos emails ?
Vous pouvez automatiser nos devis ?
Vous pouvez nous installer un assistant IA ?
```

Réponse immédiate à donner :

```txt
Oui, c’est possible de l’étudier, mais ce n’est pas inclus dans l’analyse à 200 €.

La prochaine étape consiste à cadrer le besoin exact, les outils concernés, les données nécessaires, les validations humaines et le niveau de risque.

Ensuite seulement, je peux vous proposer un devis séparé.
```

---

## Étape 1 — Vérifier si la demande est acceptable

Avant tout devis, vérifier :

- la tâche est-elle précise ?
- la tâche est-elle répétitive ?
- la tâche est-elle assez fréquente ?
- le client a-t-il des exemples ?
- le risque est-il maîtrisable ?
- un humain peut-il valider ?
- l’IA peut-elle préparer sans décider seule ?
- faut-il accéder à des données sensibles ?
- y a-t-il un outil interne à connecter ?

Refuser ou repousser si :

- le client veut une IA autonome sans contrôle ;
- le client veut que l’IA prenne des décisions juridiques, comptables, médicales, RH ou financières ;
- le client veut donner des mots de passe sans cadre sécurisé ;
- le client veut connecter des données sensibles sans contrat clair ;
- la tâche est trop vague ;
- le client attend une garantie de résultat.

---

## Étape 2 — Questions de cadrage

Questions à poser :

1. Quelle tâche exacte voulez-vous traiter ?
2. Combien de fois par semaine ou par mois cette tâche revient-elle ?
3. Qui fait cette tâche aujourd’hui ?
4. Combien de temps cela prend-il ?
5. Quel outil est utilisé aujourd’hui ?
6. Où arrivent les informations ? Email, formulaire, PDF, téléphone, Excel, logiciel métier ?
7. Quel résultat l’IA doit-elle préparer ?
8. Qui valide avant envoi, paiement, décision ou action client ?
9. Qu’est-ce que l’IA ne doit jamais faire seule ?
10. Avez-vous 5 à 10 exemples non sensibles pour tester ?

---

## Étape 3 — Classer la demande

Classer la demande dans l’un des niveaux suivants.

### Niveau 1 — Agent simple sans connexion

Exemples :

- assistant de rédaction ;
- modèle de prompt ;
- procédure IA ;
- checklist de validation ;
- trame de réponse ;
- agent utilisé manuellement par le client.

Risque : faible à moyen.

Livrable possible :

```txt
prompt maître
procédure d’utilisation
exemples avant / après
checklist de validation humaine
mini formation d’usage
```

### Niveau 2 — Agent semi-automatisé

Exemples :

- préparation de relances ;
- tri d’emails ;
- résumé de documents ;
- préparation de devis ;
- extraction d’informations simples.

Risque : moyen.

Livrable possible :

```txt
workflow cadré
outil no-code ou semi-automatisé
prompts
règles de contrôle
jeu de test
procédure de supervision
```

### Niveau 3 — Agent connecté à des outils

Exemples :

- connexion email ;
- connexion Drive ;
- connexion CRM ;
- connexion tableur ;
- connexion logiciel métier ;
- automatisation avec API.

Risque : moyen à élevé.

À ne faire que si :

- le périmètre est clair ;
- le client accepte les limites ;
- les accès sont gérés proprement ;
- les données sont nécessaires ;
- un devis séparé est signé ;
- un test sur données non sensibles est possible.

### Niveau 4 — Demande à refuser au départ

Exemples :

- paiement automatique ;
- décision commerciale sans validation ;
- envoi automatique de devis sans contrôle ;
- réponse juridique, médicale, fiscale ou RH autonome ;
- suppression ou modification automatique de données clients ;
- accès à des données sensibles sans cadre contractuel.

Réponse :

```txt
Je ne recommande pas de commencer par ce niveau d’automatisation.
On peut plutôt cadrer une version supervisée où l’IA prépare et où l’humain valide.
```

---

## Étape 4 — Pré-devis

Avant d’envoyer un prix, définir :

- objectif exact ;
- tâche concernée ;
- données utilisées ;
- outils concernés ;
- livrables ;
- délai ;
- limites ;
- responsabilités client ;
- responsabilités ScanAdmin ;
- conditions de test ;
- conditions de validation.

Ne pas envoyer un devis si la tâche n’est pas claire.

---

## Étape 5 — Structure du devis IA

Le devis doit contenir :

```txt
Prestataire : TimeProofs / ScanAdmin
Client : [entreprise]
Objet : cadrage et mise en place d’un agent IA supervisé
Périmètre exact
Ce qui est inclus
Ce qui est exclu
Délai cible
Prix
Modalités de paiement
Livrables
Limites de responsabilité
Validation humaine obligatoire
Absence de garantie de gain
Conditions d’accès aux données
```

---

## Étape 6 — Livrables possibles

Selon le niveau, les livrables peuvent être :

- fiche de cadrage ;
- cartographie de la tâche ;
- prompt maître ;
- modèle de réponse ;
- procédure d’utilisation ;
- checklist de validation humaine ;
- prototype d’agent ;
- scénario de test ;
- tableur de suivi ;
- documentation d’utilisation ;
- session de prise en main ;
- recommandations d’évolution.

---

## Étape 7 — Phase test obligatoire

Avant toute généralisation :

```txt
5 à 10 exemples réels non sensibles
→ test de l’agent
→ vérification humaine
→ correction
→ nouveau test
→ décision de continuer ou non
```

Critères à mesurer :

- temps gagné ;
- erreurs détectées ;
- corrections nécessaires ;
- cas où l’IA ne comprend pas ;
- risque d’envoi ou décision incorrecte ;
- facilité de supervision ;
- acceptation par l’équipe.

---

## Étape 8 — Mise en production prudente

Un agent IA ne doit pas être mis en production en mode autonome dès le départ.

Règles :

- démarrer sur un périmètre réduit ;
- garder une validation humaine ;
- journaliser les erreurs ;
- corriger les prompts et procédures ;
- ne pas connecter plus d’outils que nécessaire ;
- ne pas traiter de données sensibles sans cadre ;
- prévoir un arrêt manuel simple.

---

## Étape 9 — Message au client si demande trop large

```txt
Votre demande est intéressante, mais elle est trop large pour être traitée proprement tout de suite.

Je vous propose de commencer par une seule tâche : [tâche].

Objectif : créer un premier agent IA supervisé qui prépare le travail, sans décider seul.

Après 7 à 14 jours de test, on pourra décider si cela vaut la peine d’élargir.
```

---

## Étape 10 — Message au client si demande acceptable

```txt
Votre demande peut être cadrée.

La prochaine étape est un devis séparé pour :
1. définir précisément l’agent IA ;
2. préparer le workflow ;
3. créer les prompts / procédures ;
4. tester sur quelques cas ;
5. livrer une méthode utilisable avec validation humaine.

Je vous prépare une proposition avec périmètre, délai, livrables et prix.
```

---

## Règle commerciale

L’analyse à 200 € peut déboucher sur une demande d’IA, mais elle ne doit pas promettre automatiquement une mise en place.

Le bon enchaînement est :

```txt
Analyse 200 €
→ recommandation d’un agent IA supervisé
→ client intéressé
→ cadrage IA
→ devis séparé
→ prototype ou mise en place limitée
→ test supervisé
→ amélioration ou arrêt
```

---

## Statuts internes recommandés

- Demande IA reçue.
- Demande IA à clarifier.
- Demande IA refusée / trop risquée.
- Cadrage envoyé.
- Devis IA à préparer.
- Devis IA envoyé.
- Devis IA accepté.
- Devis IA refusé.
- Prototype IA en cours.
- Test IA en cours.
- Agent IA livré.
- Suivi post-livraison.

---

## Principe final

```txt
ScanAdmin ne vend pas une IA autonome.
ScanAdmin vend une IA utile, cadrée, supervisée et vérifiable.
```
